Upload-First Dataset Workflow

Overview

  • Completion of this workflow will result in a published dataset with a single resource
  • The same workflow pages will be used for adding a new dataset and editing an existing one
  • When a dataset author opts to 'Save and exit' before publishing a dataset ...
    • they land on their profile page with the dataset they were working on listed as a 'Draft'
    • when they are ready to return to editing a 'Draft' dataset, they select it from the list in their 'Profile' and should land on the first workflow page containing incomplete data
  • Authors can navigate freely between completed workflow pages in a draft dataset
  • When editing (as opposed to creating) a dataset, the check marks that indicate progress in the navigation bar return to being numbers
  • When creating a dataset, an activity stream event should be logged upon dataset publication
  • When editing a dataset, an activity stream event should be logged for any and all field updates
